# 示例文件说明

本目录包含一个完整的 PRD 示例，展示如何使用 PRD Creator Toolkit 创建高质量的产品需求文档。

## 文件列表

- `示例PRD.md` - 完整的产品需求文档（已脱敏）
- `prototypes/` - 4个交互式 HTML 原型

## 示例说明

这个示例基于一个真实的产品规划案例，但已经过脱敏处理：

### 脱敏处理

1. **公司信息**: "��时小多" → "社交创作平台"
2. **具体数据**: 保留数量级，隐藏精确数字
3. **收入数据**: 使用"基准"和"倍数"代替具体金额
4. **用户数据**: 使用"350万+"代替精确数字

### 通用化处理

1. **产品名称**: 改为通用的"像素创作玩法"
2. **功能名称**: "拼豆" → "像素创作"
3. **货币名称**: "钻石" → "积分"
4. **保留核心逻辑**: 商业化设计、诱单场景等

## 如何使用

### 1. 查看示例

用支持 HTML 的 Markdown 阅读器打开 `示例PRD.md`：

```bash
# VS Code
code 示例PRD.md

# Typora
open 示例PRD.md

# 或直接用浏览器打开（需要转换为 HTML）
```

### 2. 体验交互式原型

文档中嵌入了4个交互式原型，你可以：
- 直接在文档中查看
- 或单独打开 `prototypes/` 目录下的 HTML 文件

### 3. 作为模板使用

复制这个示例，替换为你自己的内容：

```bash
cp 示例PRD.md ../your-project/your-prd.md
cp -r prototypes ../your-project/
```

## 原型说明

### 01-什么是拼豆.html
- **用途**: 概念介绍和市场数据展示
- **特点**: 流程对比、像素演示、数据卡片
- **尺寸**: 自适应

### 02-稀有豆子系统.html
- **用途**: 核心功能和特效展示
- **特点**: CSS3 动画、普通vs VIP对比
- **尺寸**: 自适应

### 03-拼豆世界专区.html
- **用途**: 界面设计展示
- **特点**: 横屏布局、完整交互流程
- **尺寸**: 1024×768

### 04-VIP诱单场景.html
- **用途**: 商业化转化场景展示
- **特点**: 4个诱单场景、横屏展示
- **尺寸**: 640×480

## 学习要点

通过这个示例，你可以学习到：

1. **文档结构**: 如何组织一个完整的 PRD
2. **数据驱动**: 如何用数据支撑你的观点
3. **商业化设计**: 如何设计VIP权益和诱单场景
4. **原型制作**: 如何制作交互式 HTML 原型
5. **文档嵌入**: 如何在 Markdown 中嵌入原型

## 效果展示

### 文字 + 原型的完美结合

```markdown
#### 功能设计

[文字描述]

**原型展示**:

<iframe src="./prototypes/xxx.html" ...></iframe>
```

这种方式的优势：
- ✅ 汇报时无需切换工具
- ✅ 老板可以直接体验交互
- ✅ 视觉冲击力强
- ✅ 说服力提升
