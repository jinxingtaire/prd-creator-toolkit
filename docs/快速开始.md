# 快速开始指南

欢迎使用 PRD Creator Toolkit！这个指南将帮助你在 10 分钟内创建第一个带交互式原型的 PRD。

## 📋 准备工作

### 1. 安装 Markdown 阅读器

选择一个支持 HTML iframe 的 Markdown 阅读器：

**推荐选项**:
- [VS Code](https://code.visualstudio.com/) + Markdown Preview Enhanced 插件
- [Typora](https://typora.io/) (付费，但体验最好)
- [Mark Text](https://marktext.app/) (免费开源)

**不推荐**:
- GitHub 在线预览（不支持 iframe）
- 普通文本编辑器

### 2. 克隆项目

```bash
git clone https://github.com/yourusername/prd-creator-toolkit.git
cd prd-creator-toolkit
```

## 🚀 10分钟快速上手

### 第1步：查看示例（2分钟）

```bash
# 用 VS Code 打开示例
code examples/示例PRD.md

# 或用 Typora 打开
open examples/示例PRD.md
```

滚动文档，你会看到：
- ✅ 文字描述
- ✅ 嵌入的交互式原型
- ✅ 动画效果

### 第2步：复制模板（1分钟）

```bash
# 创建你的项目目录
mkdir my-prd
cd my-prd

# 复制模板
cp ../templates/PRD模板.md ./我的产品PRD.md
cp -r ../templates/prototypes ./
```

### 第3步：填写内容（5分钟）

打开 `我的产品PRD.md`，按照模板填写：

```markdown
## 一、核心结论

**战略转型**: [你要做什么？]
**商业化路径**: [怎么赚钱？]
**目标**: [预期结果？]
```

### 第4步：制作原型（可选，2分钟）

如果需要原型，编辑 `prototypes/` 下的 HTML 文件：

```html
<!-- prototypes/01-my-feature.html -->
<!DOCTYPE html>
<html>
<head>
    <title>我的功能</title>
    <style>
        /* 添加样式 */
    </style>
</head>
<body>
    <!-- 添加内容 -->
</body>
</html>
```

### 第5步：嵌入原型（1分钟）

在 Markdown 文档中嵌入原型：

```markdown
**原型展示**:

<iframe src="./prototypes/01-my-feature.html"
        width="100%"
        height="900px"
        frameborder="0"
        style="border: 1px solid #ddd; border-radius: 8px; margin: 20px 0;">
</iframe>
```

## 🎨 进阶技巧

### 添加 CSS3 动画

```css
/* 渐变动画 */
@keyframes gradient-rotate {
    0%, 100% { filter: hue-rotate(0deg); }
    50% { filter: hue-rotate(30deg); }
}

.gradient-animation {
    animation: gradient-rotate 3s ease infinite;
}

/* 闪光动画 */
@keyframes sparkle {
    0%, 100% { box-shadow: 0 0 0 rgba(255, 217, 61, 0); }
    50% { box-shadow: 0 0 30px rgba(255, 217, 61, 0.8); }
}

.sparkle-animation {
    animation: sparkle 1.5s ease infinite;
}
```

### 调整 iframe 高度

如果原型显示不完整，调整 `height` 参数：

```markdown
<!-- 高度太小，内容被截断 -->
<iframe ... height="500px"></iframe>

<!-- 调整为合适的高度 -->
<iframe ... height="1000px"></iframe>
```

**高度参考**:
- 概念介绍: 1000-1100px
- 功能展示: 900-1000px
- 界面设计: 800-900px
- 诱单场景: 1200-1400px

### 横屏 vs 竖屏

根据汇报场景选择：

```css
/* 手机竖屏 */
.phone-frame {
    width: 375px;
    height: 812px;
}

/* 平板横屏（推荐用于汇报） */
.tablet-frame {
    width: 1024px;
    height: 768px;
}

/* 展示用横屏 */
.demo-frame {
    width: 640px;
    height: 480px;
}
```

## 📊 完整工作流

### 1. 需求理解（10分钟）

回答这��问题：
- 汇报对象是谁？（老板/团队/投资人）
- 汇报时长？（5分钟/20分钟/1小时）
- 核心目的？（要资源/要决策/要支持）
- 是否需要原型？

### 2. 数据收集（30分钟）

收集这些数据：
- 内部数据：用户行为、转化率、收入
- 外部数据：市场规模、竞品数据
- 对标产品：3-5个对标产品分析

### 3. 文档撰写（2小时）

按照模板填写：
1. 核心结论（10分钟）
2. 数据支撑（30分钟）
3. 系统设计（1小时）
4. 执行计划（20分钟）

### 4. 原型制作（2-4小时）

制作3-5个原型：
1. 概念介绍原型
2. 功能展示原型
3. 界面设计原型
4. 诱单场景原型

### 5. 审阅优化（30分钟）

检查清单：
- [ ] 核心结论清晰？
- [ ] 数据充分支撑？
- [ ] 原型直观展示？
- [ ] 商业化可行？
- [ ] 执行计划明确？

## 🎯 常见问题

### Q: iframe 不显示怎么办？

**A**: 检查以下几点：
1. 文件路径是否正确？
2. 使用的 Markdown 阅读器是否支持 iframe？
3. 尝试用浏览器直接打开 HTML 文件测试

### Q: 如何导出为 PDF？

**A**:
```bash
# 使用 Typora
文件 → 导出 → PDF

# 使用 VS Code + Markdown PDF 插件
右键 → Markdown PDF: Export (pdf)

# 使用 Pandoc
pandoc 我的产品PRD.md -o 我的产品PRD.pdf
```

### Q: 可以用在 PPT 里吗？

**A**: 可以，两种方式：
1. 截图插入 PPT
2. 汇报时单独打开 HTML 文件展示

### Q: 如何分享给不懂技术的人？

**A**:
1. 导出为 PDF（推荐）
2. 或者把整个文件夹打包发送
3. 或者部署到 GitHub Pages
