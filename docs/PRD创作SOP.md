# PRD 创作 SOP

## 一、适用场景

当用户需要创建产品需求文档（PRD）或战略规划汇报时，使用此 SOP 指导完成高质量文档。

## 二、核心原则

1. **数据驱动** - 用数据说话，不拍脑袋
2. **商业化优先** - 重点讲清楚怎么赚钱
3. **可视化呈现** - 用原型、图表增强说服力
4. **结论先行** - 开门见山，先说结论
5. **精简高效** - 适合投屏，去掉冗余信息

## 四、原型制作 SOP

### 1. 确定需要的原型类型

根据 PRD 内容，确定需要哪些原型：
- **概念介绍原型** - 解释新概念、市场数据
- **功能展示原型** - 展示核心功能和交互
- **界面设计原型** - 展示完整界面布局
- **诱单场景原型** - 展示商业化转化场景

### 2. 原型设计原则

**技术选择**:
- 使用纯 HTML5 + CSS3 + JavaScript
- 无需任何依赖，直接浏览器打开
- 支持 CSS3 动画效果

**尺寸规范**:
- 手机竖屏: 375×667 或 375×812
- 手机横屏: 667×375
- 平板横屏: 1024×768 或 640×480（展示用）

**视觉规范**:
- 使用渐变色增强视觉冲击力
- 添加动画效果（渐变、闪光、发光等）
- 保持统一的设计语言

### 3. 原型文件组织

```
项目目录/
├── 需求文档/
│   ├── [产品名]战略规划汇报.md  # 主文档
│   ├── 原型/
│   │   ├── 01-[功能名].html
│   │   ├── 02-[功能名].html
│   │   ├── 03-[功能名].html
│   │   └── README.md  # 原型使用说明
│   └── 拼豆配图说明.md  # 配图文字描述（备用）
```

### 4. 嵌入主文档

在主文档中使用 iframe 嵌入原型：

```html
<iframe src="./原型/[文件名].html"
        width="100%"
        height="[高度]px"
        frameborder="0"
        style="border: 1px solid #ddd; border-radius: 8px; margin: 20px 0;">
</iframe>
```

**高度参考**:
- 概念介绍: 1000-1100px
- 功能展示: 900-1000px
- 界面设计: 800-900px
- 诱单场景: 1200-1400px

## 五、创作流程

### 阶段1：需求理解（10分钟）

1. **明确汇报对象**
   - 老板/投资人 → 重点讲商业化
   - 团队/开发 → 重点讲功能细节
   - 用户/市场 → 重点讲价值和体验

2. **明确汇报目的**
   - 要资源 → 强调投入产出比
   - 要决策 → 强调风险和收益
   - 要支持 → 强调战略意义

3. **明确时间限制**
   - 5分钟 → 只讲核心结论和数据
   - 20分钟 → 完整讲述，重点突出
   - 1小时 → 详细讲述，包含细节

### 阶段2：数据收集（30分钟）

1. **内部数据**
   - 用户行为数据
   - 转化率数据
   - 收入数据

2. **外部数据**
   - 市场规模（微信指数、百度指数）
   - 竞品数据（小红书、淘宝）
   - 行业报告

3. **对标产品**
   - 找3-5个对标产品
   - 分析其商业模式
   - 提取可借鉴的点

### 阶段3：文档撰写（2小时）

1. **先写结论**（10分钟）
   - 用一句话说清楚要做什么
   - 用一句话说清楚怎么赚钱
   - 用一句话说清楚预期结果

2. **再写数据**（30分钟）
   - 用数据证明问题存在
   - 用数据证明方案可行
   - 用数据证明收益可观

3. **然后写方案**（1小时）
   - 系统设计
   - 功能设计
   - 商业化设计

4. **最后写执行**（20分钟）
   - 时间线
   - 里程碑
   - 风险应对

### 阶段4：原型制作（2-4小时）

1. **确定原型数量**（10分钟）
   - 一般3-5个原型即可
   - 重点展示核心功能和商业化场景

2. **制作原型**（每个30-60分钟）
   - 使用 HTML5 模板
   - 添加动画效果
   - 测试浏览器兼容性

3. **嵌入文档**（10分钟）
   - 在合适位置添加 iframe
   - 调整高度适配
   - 测试显示效果

### 阶段5：审阅优化（30分钟）

1. **自查清单**
   - [ ] 结论是否清晰？
   - [ ] 数据是否充分？
   - [ ] 逻辑是否严密？
   - [ ] 原型是否直观？
   - [ ] 商业化是否可行？

2. **优化方向**
   - 删除冗余信息
   - 增强视觉呈现
   - 简化复杂表述
   - 突出核心亮点

## 六、常见问题

### Q1: 如何快速找到市场数据？

**A**:
- 微信指数: https://index.weixin.qq.com
- 百度指数: https://index.baidu.com
- 小红书: 直接搜索关键词，看笔记数量
- 淘宝: 搜索关键词，看销量排名

### Q2: 如何设计有说服力的商业化方案？

**A**:
1. 对标成功产品的商业模式
2. 用数据推算收入（保守估计）
3. 设计多个收入来源（不要单一）
4. 场景化诱单（在用户最想要的时刻转化）

### Q3: 原型一定要做吗？

**A**:
- 如果是给老板汇报 → 强烈建议做，视觉冲击力强
- 如果是给团队看 → 可以简化，重点讲功能
- 如果时间紧 → 至少做1-2个核心原型

### Q4: iframe 不显示怎么办？

**A**:
1. 检查文件路径是否正确
2. 用支持 HTML 的 Markdown 阅读器（VS Code、Typora）
3. 或者改用截图方式
4. 或者汇报时单独打开 HTML 文件

### Q5: 如何估算收入数据？

**A**:
1. 找到关键指标（DAU、转化率、客单价）
2. 参考对标产品的数据
3. 保守估计（打7折）
4. 分阶段预测（不要一步到位）
